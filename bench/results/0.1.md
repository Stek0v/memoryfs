## Conclusion

Architecture comfortably handles 100k files. Read p95 is 260x under the SLO
target. Write is I/O-bound (SHA-256 + atomic file write) at ~3.2k ops/s.
The sharded object store layout (`aa/bb/hash`) avoids filesystem bottlenecks.

Diff on 100k changes is the slowest operation at ~8ms (BTreeMap comparison).
This is acceptable for the commit-graph use case.
